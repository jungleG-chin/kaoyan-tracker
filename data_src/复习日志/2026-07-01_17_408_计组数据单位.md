# 408 计组数据单位

## K_计组数据单位 · 408 · 数据单位换算（K/M/G/T/P/E/Z） [warn · 5/10]

- **章节**：【CO】一、计算机系统概述
- **题目**：计组大纲要求掌握的数据单位 K、M、G、T、P、E、Z 的换算关系？
- **你的回答**：需要复习
- **缺漏**：需要强化记忆 1024 倍关系（2¹⁰），以及磁盘/内存/网络常见容量单位的对应
- **标准答案**：【数据单位换算】（2¹⁰ 倍数）

K = 2¹⁰ = 1024
M = 2²⁰ = 1024² ≈ 10⁶
G = 2³⁰ = 1024³ ≈ 10⁹
T = 2⁴⁰ = 1024⁴ ≈ 10¹²
P = 2⁵⁰ = 1024⁵ ≈ 10¹⁵
E = 2⁶⁰ = 1024⁶ ≈ 10¹⁸
Z = 2⁷⁰ = 1024⁷ ≈ 10²¹

【考试常考点】
1. 内存容量、磁盘容量、Cache 容量都按 2¹⁰ 算
2. 传输速率单位是 bit/s（b/s），容量单位是 Byte（B），1B = 8b
3. 常见大题：地址位数 = ⌈log₂(容量)⌉，例如 4GB 内存需要 32 位地址
4. 速算口诀：2¹⁰ ≈ 10³，2²⁰ ≈ 10⁶，2³⁰ ≈ 10⁹
- **错误**：否

## K_多级页表大小限制 · 408 · 多级页表：顶级页表 vs 内部页表的大小限制 [pass · 8/10]

- **章节**：【OS】三、内存管理
- **题目**：多级页表中，为什么顶级页表可以超过一个页大小，而其他级别的页表不能？
- **你的回答**：顶级页表支持超过一页大小，因为底层硬件支持；其他多级页表不支持单个页面超过一页大小
- **缺漏**：理解方向正确，但需要补充「为什么」的底层原因：① 顶级页表有专用基址寄存器（如 x86 的 CR3），始终驻留内存，无需被换入换出，所以可以跨多页；② 内部各级页表必须恰好占一页，是因为它们本身也要受分页机制管理——只有占一页，才能被换入换出，由上一级页表项指向其所在页框。
- **标准答案**：【核心结论】
用户总结基本正确，补充底层原因：

1. 顶级页表（页目录/外层页表）：
   - 可以超过一个页的大小
   - 原因：有专用基址寄存器（如 x86 的 CR3）指向它，始终驻留内存
   - 不需要被换入换出，所以没有「必须占一页」的限制

2. 内部各级页表（二级、三级…）：
   - 必须恰好占一个页框
   - 原因：它们本身要受分页机制管理，需要被换入换出
   - 只有占一页，才能被上一级页表项直接指向其所在页框号
   - 如果超过一页，就无法用一个页表项定位

【一句话记忆】
顶级页表有「特权」（专用寄存器 + 常驻内存），可以跨页；内部页表是「普通人」，必须占一页才能被管理。
- **错误**：否

## K_OS操作并行vs串行计时 · 408 · OS时间计算：操作并行 vs 串行的细节 [pass · 8/10]

- **章节**：【OS】三、内存管理
- **题目**：计算两个操作的总时间时，如何判断它们是并行还是串行？
- **你的回答**：有时两个操作是并行的（同时进行），有时是一个操作完成后另一个才开始，计算总时间时要注意区分
- **缺漏**：总结方向正确，补充常见场景判断规则
- **标准答案**：【核心要点】
计算操作总时间时，必须先判断并行还是串行：

1. **并行（同时进行）**：总时间 = max(t₁, t₂)
   - 例：磁盘寻道 + 旋转等待可以重叠（一旦开始寻道，磁盘也在转）
   - 例：DMA传输与CPU计算可并行

2. **串行（先后进行）**：总时间 = t₁ + t₂
   - 例：缺页处理中，先从磁盘读入页面，再更新页表，这两个步骤不可重叠
   - 例：先查找目录项，再读数据块

【判断原则】
- 两个操作依赖同一硬件资源 → 通常串行
- 两个操作使用不同硬件/可独立推进 → 可能并行
- 题目明确说「同时」「期间」 → 并行
- 题目明确说「然后」「之后再」 → 串行

【易错点】
磁盘访问时间 = 寻道时间 + 旋转等待时间 + 传输时间
→ 但寻道和旋转可以重叠！实际 = max(寻道, 旋转) + 传输

一句话：算时间先看「能不能同时干」，能就取max，不能就相加。
- **错误**：否
