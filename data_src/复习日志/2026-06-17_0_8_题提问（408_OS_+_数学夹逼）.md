# 8 题提问（408 OS + 数学夹逼）

## Q1 · 408 · OS 运行模式 [pass · 8/10]

- **章节**：【OS】一、操作系统概述与运行环境
- **题目**：操作系统运行环境里，CPU 有哪两种运行模式？它们的核心区别是什么？
- **你的回答**：用户态和内核态两种模式。内核态是对所有指令有完全的权限，用户态则对某些指令有权限。这就是确保操作系统的安全。
- **标准答案**：✓ 标准答案：
CPU 两种模式：内核态（Kernel Mode）和用户态（User Mode）。
核心区别：内核态可执行特权指令，用户态只能执行普通指令。
为什么需要：保护操作系统内核和关键资源不被用户程序破坏。
补充：用户态→内核态的切换途径：①系统调用 ②中断 ③异常。
- **错误**：否

## Q2 · 408 · 进程 vs 线程 [fail · 5/10]

- **章节**：【OS】二、进程管理
- **题目**：进程和线程最本质的区别是什么？一个进程里的多个线程是否共享堆栈？
- **你的回答**：进程是资源分配的基本单位，线程是调度的基本单位。进程里的堆栈应该是可以修改的，那我认为多个线程应该是可以共享堆栈的。
- **缺漏**：线程共享的是堆（heap），不共享栈（stack）。栈是线程私有的。
- **标准答案**：✓ 标准答案：
- 进程是资源分配的基本单位，线程是 CPU 调度的基本单位。
- 同一进程内的多个线程共享：堆、全局变量、代码段、文件描述符等。
- 线程私有：栈（stack）、程序计数器、寄存器组。
- 错点：线程不共享栈！
- **错误**：是

## Q4 · 408 · TSL 指令 [fail · 5/10]

- **章节**：【OS】二、进程管理
- **题目**：TSL 指令的原子性是怎么保证的？它和 Peterson 算法相比，最大的优势是什么？
- **你的回答**：原子性是通过这个进程进来之后先读取也就是 test 一下锁。如果不等于 0 就等待直到锁等于 0 再进去。优势应该是忙着等待吧。
- **缺漏**：TSL 也是忙等，'不忙等'不是优势。真正优势：硬件原子性，可在多 CPU 上正确工作。
- **标准答案**：✓ 标准答案：
- 原子性由硬件实现，TSL 是一条单条指令，从读取到设置 lock 不可被中断。
- 优势：硬件原子性 + 多 CPU 安全。Peterson 在多 CPU 下会失效。
- TSL 也是忙等！'不忙等'不是优势。
- **错误**：是

## Q6 · 数学二 · 夹逼定理使用条件 [fail · 4/10]

- **章节**：一、函数、极限、连续
- **题目**：夹逼定理的两个使用前提是什么？
- **你的回答**：前提是他大于和小于的那两个数的极限必须相等，还有一个我不记得了，难道是连续吗？
- **缺漏**：缺核心条件'小 ≤ 目标 ≤ 大'（夹在中间）；不要求连续。
- **标准答案**：✓ 标准答案：
夹逼定理三件套：
① 存在性：能找到两个数列 a_n、b_n，使得 a_n ≤ 目标 c_n ≤ b_n。
② 同极限：lim a_n = lim b_n = L。
③ 结论：则 lim c_n = L。
不要求连续、不要求单调。

## Q7 · 数学二 · 定积分定义识别 [warn · 7/10]

- **你的回答**：我会用定积分定义，因为有一个可爱因子，然后很明确 X 是从 0 到 1……啊不是从 0 到 π。
- **缺漏**：没说清楚定积分定义的判据：'Σ f(xk)·Δx' 形式 + Δx→0 + xk 走遍 [a,b]。
- **标准答案**：✓ 标准答案：

③ x_k = kπ/n 走遍 [0, π]。
所以原式 = ∫₀^π sin x dx = 2。

## Q8 · 数学二 · n项求和 + 高次量级 [fail · 0/10]

- **章节**：一、函数、极限、连续
- **题目**：lim(n→∞) Σ(k=1 to n) k²/√(n⁶+k²)，用夹逼怎么处理放缩上界？
- **你的回答**：完全不知道怎么没头绪。
- **缺漏**：会写：k²/√(n⁶+k²) ≤ k²/√(n⁶) = k²/n³；Σ 上界 = (1/n³)·Σk² = (1/n³)·n(n+1)(2n+1)/6 → 0。
- **标准答案**：✓ 标准答案：
夹逼三步：
① 上界放缩：k²/√(n⁶+k²) ≤ k²/√(n⁶) = k²/n³
   Σ 上界 = (1/n³)·Σk² = (1/n³)·n(n+1)(2n+1)/6 → 0
② 下界放缩：k²/√(n⁶+n⁶) = 1/(2√2 n²)
   Σ 下界 ≥ n·1/(2√2 n²) = 1/(2√2 n) → 0
③ 上下界都 → 0，所以原极限 = 0。
- **错误**：是
