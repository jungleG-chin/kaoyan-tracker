# 408 · 指令系统-有效地址本质

## K_PC和跳转指令中的虚拟地址 · 408 · 指令系统中的地址概念 [pass · 10/10]

- **章节**：【CO】四、指令系统
- **题目**：CPU 算出的有效地址是什么？PC 寄存器存的是什么？jump 指令的跳转目标是什么地址？
- **思路**：
  1. **CPU 算出的有效地址 = 虚拟地址（逻辑地址），不是物理地址**
  2. **PC 寄存器存的也是虚拟地址**（下一条要执行的指令的虚拟地址）
  3. **jump/branch 指令的跳转目标地址 = 虚拟地址 ± 偏移量**（PC相对寻址的偏移加在虚拟地址上）
  4. 物理地址的转换由 **MMU（内存管理单元）** 在访存阶段完成
  5. 整个流程：指令中的地址字段 / PC → CPU 算出有效地址（虚拟地址）→ MMU 查页表 → 物理地址 → 访存
- **关键区分**：
  - 程序员/CPU 看到的地址 = **虚拟地址**
  - 实际访问内存用的地址 = **物理地址**
  - PC 存的、jmp 跳转的、CPU 算出的 **都是虚拟地址**
  - 转换过程对程序员透明，由硬件+OS 协作完成
- **考试常考**：
  - "有效地址"通常指**虚拟地址**
  - 问"PC 存什么"——答：下一条指令的虚拟地址
  - 跳转指令的偏移量是**相对 PC 的虚拟地址偏移**
  - 在分页/分段系统中，CPU 给出的都是虚拟地址
- **错误**：否
