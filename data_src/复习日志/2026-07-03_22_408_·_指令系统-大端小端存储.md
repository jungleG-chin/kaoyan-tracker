# 408 · 计组 · 指令系统-大端/小端存储

## K_大端小端存储 · 408 · 指令系统 · 边界对齐与存放方式 [掌握]
- **章节**：【CO】四、指令系统
- **题目**：复习数据存放方式，对比大端存储 vs 小端存储
- **你的理解**：
  - 大端存储 = 更符合人类视角
  - 小端存储 = 大端倒过来
  - 最小块是字节(8 bit)，按图示的字节粒度排列
- **思路/方法论**：
  - **大端(Big-Endian)**：数据的**最高有效字节(MSB)**存放在**最低内存地址**处。
    - 例：int 1 = 0x00 00 00 01，存到地址递增方向 → `00 00 00 01`（高位在低地址）
  - **小端(Little-Endian)**：数据的**最低有效字节(LSB)**存放在**最低内存地址**处。
    - 同样 int 1 = 0x00 00 00 01，小端存为 → `01 00 00 00`（低位在低地址）
  - **核心判断口诀**：
    - 大端 = "高对低"（高位字节放低地址）
    - 小端 = "低对低"（低位字节放低地址）
  - **最小单位**：按字节编址（1 Byte = 8 bit），图中的每个小块 = 1字节
  - **常见处理器**：
    - x86 / x86_64 → 小端
    - ARM / MIPS → 可配置（默认通常大端或小端，依实现）
    - 网络协议(TCP/IP) → 大端（也称"网络字节序"）
  - **常见考法**：
    1. 给一个十六进制数，问在大端/小端机器上内存如何排布
    2. 跨平台通信时为什么要做字节序转换
    3. 大小端对位域、结构体内存布局的影响
- **关键记忆点**：
  - 区分"字节序" vs "位序"：本图讨论的是**字节之间的顺序**，不是位之间的顺序
  - 大端符合直觉（人读数从左到右 = 地址从小到大）
  - 小端优点：低地址取低字节，做加减法、类型转换更高效（x86 选小端的原因）
- **考试常考形式**：
  - 选择题：给一个十六进制数 0x12345678，画出大端和小端在内存中的排布
  - 综合题：跨平台数据交互时给数据，问对方机器读到的值是什么
  - 概念辨析：大端 vs 小端 vs 按边界对齐（不按边界对齐 vs 按边界对齐是另一组分类）
