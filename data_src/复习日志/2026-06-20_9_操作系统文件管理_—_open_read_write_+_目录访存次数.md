# 操作系统文件管理 — open/read/write + 目录访存次数

## K_文件IO与目录访存 · 408 · 文件系统：open/read/write 系统调用 + 目录结构访存计算 [pass · 8/10]

- **章节**：【OS】四、文件管理
- **题目**：操作系统如何通过 FD（文件描述符）进行文件读写？多级目录下打开一个文件需要几次磁盘访问？
- **你的回答**：open得到FD → read/write操作这个FD变量相当于生成文件副本 → write修改副本后覆盖源文件。三级目录下open需要访存5次。
- **标准答案**：【一、文件I/O完整流程】

Step 1: fd = open("路径", 模式)
  - 功能：打开文件，返回文件描述符FD（一个整数，进程级的索引）
  - 内部做了什么：
    ① 根据文件路径名，从根目录开始逐级查找
    ② 找到文件的FCB（文件控制块/ inode）
    ③ 将FCB信息复制到内存的**打开文件表**
    ④ 返回fd（指向该表项的指针）

Step 2: read(fd, buf, n)
  - 功能：从fd对应的文件读n个字节到buf缓冲区
  - 本质：根据fd找到内存中的打开文件表项 → 找到当前读写位置 → 从磁盘读数据到buf
  - 【注意】read不是直接改源文件！是从磁盘读到用户空间的buf里

Step 3: write(fd, buf, n)
  - 功能：把buf里的n个字节写入fd对应的文件
  - 本质：buf数据 → 写入内核缓冲区（延迟写/立即写取决于策略）→ 最终回写到磁盘
  - 【注意】write不是覆盖整个文件！是从当前位置开始写入指定字节数

【二、FD的本质 — 重要澄清】

你的理解：'FD = 文件副本'
更准确的说法：
- FD是一个整数索引，指向进程的**打开文件表**中的某个条目
- 打开文件表中存的是：文件位置指针、访问权限、FCB副本等元数据
- **不包含文件内容本身！** 文件内容还在磁盘上
- read/write时才真正访问磁盘上的数据块

【三、目录结构下的访存次数计算】

假设三级目录：/A/B/C/file.txt
目录层级:  [根目录] → [一级目录A] → [二级目录B] → [三级目录C] → [文件]

open("/A/B/C/file.txt") 的磁盘访存过程:

  访存1: 读根目录(在内存中可能命中, 假设不在)
  访存2: 读一级目录 A
  访存3: 读二级目录 B
  访存4: 读三级目录 C
  访存5: 读文件的 FCB (inode)

**总计: 5次磁盘访问** ✓ (你说的对!)

通用公式: N级目录 + 1个文件 = N+1次访存
(假设每级都不在缓存中，且不考虑FCB缓存优化)

【四、考试高频考点】

1. FD vs 文件指针:
   - FD: 进程级，每个进程有自己的fd表
   - 系统级打开文件表: 全局共享，多个进程可以打开同一文件

2. open做了什么(选择题常考):
   - 权限检查
   - 目录搜索(找FCB)
   - 分配fd
   - 初始化读写位置为0

3. close的作用:
   - 释放fd(可复用给其他open)
   - 如果是最后一个引用，释放系统级打开文件表项
   - 将脏页写回磁盘

4. 缓冲区问题:
   - write的数据先到内核缓冲区，不一定立刻落盘
   - 断电可能丢失未flush的数据
   - fsync() 强制刷盘
- **错误**：否
